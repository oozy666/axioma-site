Privacy Policy

This Privacy Policy describes how Mailer by Axioma ("the Application") accesses, uses, stores, and protects user data.

1. Application Overview

Mailer by Axioma is a Telegram-based email automation tool that allows users to send emails from their own Google accounts using the Gmail API.

2. Google User Data Accessed

The Application accesses the following Google user data via Google OAuth 2.0:

  • Gmail API (https://www.googleapis.com/auth/gmail.send)
    Used solely to send emails on behalf of the authenticated user. The Application does not read, access, or analyze incoming emails, attachments, or mailbox contents.
  • Google Drive API (https://www.googleapis.com/auth/drive.file)
    Used only to access a spreadsheet file explicitly provided by the user, containing email recipient addresses for an email campaign. The Application does not access or analyze any other files on the user's Google Drive.

3. Data Provided Directly by the User

  • Telegram user ID and username
  • Email campaign content (subject and message body)
  • Email recipient list (provided via a Google Spreadsheet)

4. How Data Is Used

User data is used exclusively to provide the core functionality of the Application, which is sending email campaigns on behalf of the user.

  • Email content is used to send messages via the Gmail API
  • Recipient lists are processed temporarily to complete a mailing campaign
  • Email campaign history (subject and message body) is stored to allow users to review previous campaigns

5. Data Storage and Security

OAuth access and refresh tokens are stored securely in an encrypted form using industry-standard encryption methods. Encryption keys are stored separately using environment variables.

Email recipient lists are downloaded temporarily to the server only for the duration of an active mailing process and are permanently deleted immediately after the campaign is completed.

Email campaign history (email subject and message body) is stored in a secure database and remains stored until user data is removed.

6. Data Sharing

The Application does not sell, rent, or share Google user data with third parties. Data is used strictly to provide the Application’s functionality.

7. Data Retention and Deletion

OAuth tokens remain stored until the user revokes access to the Application. Users can revoke access at any time through their Google Account permissions page:

https://myaccount.google.com/permissions

Once access is revoked, the stored OAuth tokens become invalid and can no longer be used to access Google services.

8. User Control

Users have full control over their Google account permissions and can revoke the Application’s access at any time through Google Account settings.
